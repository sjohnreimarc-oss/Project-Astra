PROJECT ASTRA DEVELOPMENT RULES

1. One major physics feature per version.
2. Every version must compile and work before moving on.
3. Commit and push before adding another feature.
4. Never implement two major equations in one sitting.
5. If a feature takes more than one day, that's okay.
6. Never rewrite working code unless necessary.
7. Resist feature creep.
8. If NASA uses more equations than you currently understand, you are not implementing them yet.
9. Educational mode is NOT required until v1.0+.

Small versions are GOOD versions.
